# 网站部署指南 - 让别人通过链接访问你的网站

## 🌐 方法一：GitHub Pages（推荐，完全免费）

### 步骤：
1. **注册 GitHub 账号**
   - 访问 https://github.com
   - 点击 Sign up 注册（如已有账号直接登录）

2. **创建新仓库**
   - 点击右上角 "+" → "New repository"
   - 仓库名填写：`uotan-toolbox-website`（或任意名称）
   - 选择 "Public"（公开）
   - 点击 "Create repository"

3. **上传文件**
   - 点击 "uploading an existing file"
   - 将 `index.html` 文件拖拽到页面
   - 点击 "Commit changes"

4. **启用 GitHub Pages**
   - 进入仓库的 Settings（设置）
   - 左侧菜单找到 "Pages"
   - Source 选择 "main" 分支
   - 点击 "Save"

5. **获取网站链接**
   - 等待 1-2 分钟
   - 页面会显示：`Your site is live at https://你的用户名.github.io/仓库名/`
   - 这就是你的网站地址！

### 示例链接：
```
https://yourname.github.io/uotan-toolbox-website/
```

## 🇨🇳 方法二：Gitee Pages（国内访问更快）

### 步骤：
1. **注册 Gitee 账号**
   - 访问 https://gitee.com
   - 点击注册（如已有账号直接登录）

2. **创建新仓库**
   - 点击右上角 "+" → "新建仓库"
   - 仓库名称：`uotan-toolbox-website`
   - 选择 "公开"
   - 点击 "创建"

3. **上传文件**
   - 进入仓库
   - 点击 "上传文件"
   - 选择 `index.html`
   - 点击 "提交"

4. **启用 Gitee Pages**
   - 点击仓库顶部的 "服务" → "Gitee Pages"
   - 点击 "启动"
   - 选择 "master" 分支
   - 点击 "启动"

5. **获取网站链接**
   - 页面会显示：`https://你的用户名.gitee.io/仓库名/`
   - 这就是你的网站地址！

### 示例链接：
```
https://yourname.gitee.io/uotan-toolbox-website/
```

---

## ⚡ 方法三：Vercel（最简单，自动部署）

### 步骤：
1. **访问 Vercel**
   - 打开 https://vercel.com
   - 点击 "Sign Up" 注册（可用 GitHub 账号登录）

2. **导入项目**
   - 点击 "Add New..." → "Project"
   - 选择 "Import Git Repository"
   - 连接你的 GitHub 账号
   - 选择刚才创建的仓库

3. **部署**
   - 点击 "Deploy"
   - 等待 30 秒

4. **获取链接**
   - 部署完成后会显示：`https://你的项目名.vercel.app`
   - 这就是你的网站地址！

### 示例链接：
```
https://uotan-toolbox-website.vercel.app
```

---

## 🚀 方法四：Netlify（功能强大）

### 步骤：
1. **访问 Netlify**
   - 打开 https://www.netlify.com
   - 点击 "Sign up" 注册

2. **拖拽部署**
   - 登录后点击 "Add new site" → "Deploy manually"
   - 将包含 `index.html` 的文件夹直接拖拽到页面
   - 等待上传完成

3. **获取链接**
   - 自动生成链接：`https://随机名称.netlify.app`
   - 可以在 Site settings 中修改域名

### 示例链接：
```
https://uotan-toolbox.netlify.app
```

---

## 📱 方法五：Cloudflare Pages（全球CDN加速）

### 步骤：
1. **访问 Cloudflare Pages**
   - 打开 https://pages.cloudflare.com
   - 注册/登录账号

2. **创建项目**
   - 点击 "Create a project"
   - 连接 GitHub 账号
   - 选择仓库

3. **部署**
   - 点击 "Begin setup"
   - 点击 "Save and Deploy"

4. **获取链接**
   - 链接格式：`https://项目名.pages.dev`

## 🎯 推荐方案对比

| 平台 | 优点 | 缺点 | 推荐指数 |
|------|------|------|----------|
| **GitHub Pages** | 完全免费、稳定、知名度高 | 国内访问稍慢 | ⭐⭐⭐⭐⭐ |
| **Gitee Pages** | 国内访问快、中文界面 | 需要实名认证 | ⭐⭐⭐⭐⭐ |
| **Vercel** | 自动部署、速度快 | 需要科学上网 | ⭐⭐⭐⭐ |
| **Netlify** | 功能强大、易用 | 国内访问一般 | ⭐⭐⭐⭐ |
| **Cloudflare** | 全球CDN、速度快 | 配置稍复杂 | ⭐⭐⭐⭐ |

## 🔧 快速部署命令（适合有 Git 基础的用户）

### GitHub Pages 快速部署：
```bash
# 1. 初始化 Git 仓库
cd website
git init

# 2. 添加文件
git add index.html

# 3. 提交
git commit -m "Initial commit"

# 4. 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/仓库名.git

# 5. 推送
git branch -M main
git push -u origin main

# 6. 在 GitHub 仓库设置中启用 Pages
```

## ❓ 常见问题

### Q1: 部署后网站打不开？
A: 等待 1-5 分钟，GitHub Pages 需要时间生成网站。

### Q2: 修改了代码如何更新网站？
A: 重新上传文件到仓库，或使用 Git 推送更新。

### Q3: 可以绑定自己的域名吗？
A: 可以！在各平台的设置中都有自定义域名选项。

### Q4: 这些服务收费吗？
A: 基础功能完全免费，足够个人使用。

### Q5: 国内用户推荐哪个？
A: **Gitee Pages** 最适合国内用户，访问速度快且稳定。
